# heimdall — docs

Transparent TLS-aware egress proxy + observability for Kubernetes pods
and arbitrary CLI processes, built on eBPF and aya.

## Reading order

1. **[architecture.md](architecture.md)** — components, the five eBPF
   programs (`connect4`, `connect6`, `udp4_sendmsg`, `udp6_sendmsg`,
   `skb_egress`), the dual-stack data flow, the policy plane, the
   `heimdall run` lifecycle, and the orphan-cgroup GC.
2. **[config.md](config.md)** — pod-side: how a Kubernetes workload
   selects a connection (annotation > label > admin rule) and the
   `routing` × `observe` model. For the full admin schema, see
   `/etc/heimdall/README.md` (auto-generated by `heimdall init`).
3. **[observability.md](observability.md)** — Phase B TLS plaintext
   tap: coverage matrix (libssl, Go, rustls, JVM), how stripped Go
   binaries are handled via `.gopclntab`, flow_id correlation, and
   why the tap layer is transport-agnostic across IPv4 / IPv6.
4. **[runbook.md](runbook.md)** — daily ops: building, deploying,
   reading the UI, capability requirements, and troubleshooting for
   tap attach failures, `heimdall run`, and GC.

## What heimdall is, in 90 seconds

A pod (or a `heimdall run`-wrapped process) opens a TCP connection.
eBPF's `connect4` / `connect6` hooks intercept it **inside the
process's cgroup** before the kernel routes the packet, look up a
per-cgroup policy in a BPF map, and either:

- Rewrite the destination to the relay (`relayIp:12345` /
  `[relayIp6]:12345`) so the daemon can take over → SOCKS5 to the
  selected upstream, with the original hostname carried via
  ATYP=0x03 (fake-IP reverse lookup) when DNS went through heimdall.
- Let the kernel route natively (`use: system`, kernel-bypass CIDRs,
  pod-internal traffic, kube-apiserver chatter, …).

DNS gets the same treatment via `udp4_sendmsg` / `udp6_sendmsg` —
when a cgroup is registered with `dns: fake`, UDP `:53` is rewritten
to heimdall's fake-IP DNS server. CLI clients additionally get a
mount-namespace shim that bypasses NixOS' nscd + nss-resolve so
`getaddrinfo` reaches the rewritten `127.0.0.1:53`.

Independently of routing, **every cgroup also has an `observe` flag**.
When on, libssl / Go TLS / rustls uprobes capture decrypted payloads
on `SSL_write` / `SSL_read` / `crypto/tls.(*Conn).{Read,Write}` /
rustls `PlaintextSink::write` and stream them through a perf event
array into a sqlite store. The Web UI shows live decrypted plaintext
alongside connection metadata.

Both axes are configurable per-rule and per-pod (annotations override
rules), so noisy infrastructure pods can be silenced while user
workloads stay observed.
